## iOS / SuiGrpcKit surface

The `SuiGrpcKit` package exposes:

- `getLatestEpoch()` — **used directly** by `ZkLoginCoordinator.swift:467`.
- `getReferenceGasPrice()` — exposed; **not called directly** from `Talise` target today (web route is used instead).
- `getBalance(address:coinType:)` — exposed; **not called directly** from `Talise` target today. Balances are read via `/api/balances` and `/api/wallet/balances` web routes.
- `executeTransaction(...)` — exposed; **not called directly** today. Execution goes through `/api/zk/sponsor-execute`.

iOS endpoints currently still going through web routes (candidates for
future direct gRPC migration):

- Balances (#1–4) → could call `SuiGrpcClient.shared.getBalance` for the
  hot-path render on Home, then reconcile via `/api/balances` for the
  vault fold-in. Would shave one HTTP hop.
- Epoch read (#22) — already exposed; second consumer (zk warmup) could
  reuse it instead of routing through `/api/sui/epoch`.

## Known broken / degraded

### ❌ #5 Activity feed — FIXED in this PR

Root cause: `fetchLimit = Math.max(limit * 4, 50)` produced
`first: 80` whenever iOS asked for `limit=20`. Sui mainnet GraphQL caps
`transactions(first:)` at 50 — it returned a `GRAPHQL_VALIDATION_FAILED`
error which the broad `try/catch` in `getRecentActivity` swallowed,
returning `[]`. Every active user with `limit=20` saw "Nothing yet".
The probe confirms 19 entries for the admin tester's address
`0x156a…3743` after the fix — pagination loop caps each page at 50 and
walks the cursor until `fetchLimit` is hit.

### ⚠ #4 `/api/wallet/balances` still JSON-RPC

`app/api/wallet/balances/route.ts:47` posts a raw
`{ jsonrpc: "2.0", method: "suix_getAllBalances" }` body to `RPC_URL`.
The migration plan in `docs/sui-rpc-migration/migration-plan.md`
declared JSON-RPC dead in Phase 5; this is a leftover. Replace with
`sui().listBalances({ owner })`. Risk is low (read-only), urgency
medium — this is the route iOS calls when rendering the wallet
balances list on the home screen.

### ⚠ #22 Duplicate epoch read paths

Both `SuiGrpcKit.getLatestEpoch()` (direct gRPC from iOS) and
`/api/sui/epoch` (web → gRPC via sui()) exist. Each call site uses one
or the other. Not broken, but inconsistent — future zk warmup work
should pick one path and delete the other.

## Latency benchmark — 2026-05-29

Five calls × 5 runs each, against the fallback-proxy `sui()` in
`lib/sui.ts`. One warmup hit before each batch so we measure
steady-state latency. Source: `web/scripts/grpc-bench-2026-05-29.mjs`
against the admin tester address.

| Read | min | p50 | p95 | max |
| ---- | ---:| ---:| ---:| ---:|
| `getSuiBalance` | 209 | **228** | 866 | 866 |
| `getUsdsuiBalance` | 190 | **239** | 267 | 267 |
| `getReferenceGasPrice` | 215 | **232** | 237 | 237 |
| `getLatestEpoch` (ledgerService) | 199 | **226** | 269 | 269 |
| `getCoinMetadata(USDsui)` | 215 | **237** | 276 | 276 |

All five p50s land in 226–239ms — within the 250ms target. Headroom is
slim and the cold-start p95 on `getSuiBalance` (866ms) crosses it.

### Proposed fixes (NOT yet implemented)

Ranked by impact on the 250ms p50 target:

1. **Warm channel at module load.** The 866ms cold p95 on
   `getSuiBalance` is the proxy template + HTTP/2 connection setup.
   `lib/sui.ts` already calls `void sui()` at module load to build the
   proxy — extend that warmup to fire a real
   `getReferenceGasPrice()` so the HTTP/2 connection (Posix transport)
   is up before the first user-facing call. Expected: drops first-call
   p95 from 866ms → ~230ms, brings p95 in line with p50.

2. **Memoize gas price + epoch with short TTL (≤2s).** Reference gas
   price changes per epoch (24h); epoch ID changes per epoch. Caching
   both for 1–2s eliminates the second wave of reads during a
   sponsor-prepare → execute flow. Expected: removes 2 × ~230ms hits
   from every send (~460ms saved end-to-end).

3. **Pre-fetch balances in handler.** `/api/balances` and
   `/api/wallet/balances` currently read serially. Issue
   `getSuiBalance + getUsdsuiBalance + getUsdcBalance` in
   `Promise.all` (they already do for some pairs; verify every
   handler does). Expected: shaves 200–300ms off the home render's
   balance read path by eliminating the third sequential await.

(Fixes 1–3 are independent and stackable. Implement in this order.)
